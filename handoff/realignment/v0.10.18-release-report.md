# Release Completion Report — v0.10.18

Source-seed fix for the 4 catalog placeholders that pre-v0.10.18 advertised in the operator's Telemetry deck but had no managed pool spec to back them. The operator's testing-phase audit reported "26 reported / 21 reachable" against a 26-element MCP server catalog. Root cause: `MasterControlRuntime.cpp`'s v0.9.64 seed batch registered 18 in-tree baseline-worker pools and 7 sub-agent pools, but did NOT register pools for the 4 (5 including sqlite) external `@modelcontextprotocol/server-*` placeholders sitting in `MasterControlDefaults.cpp`'s catalog. v0.10.18 closes the gap with a `registerNpxMcpPool` seed lambda.

## 1. Operator-reported defect

**Surface:** WinUI Shell Telemetry deck.
**Symptom:** "26 reported MCP servers / 21 reachable."
**Verified live (pre-fix):** 21 of 26 — 18 in-tree baseline-worker pools + 3 in-process governance servers = 21 `reachable=true`. The other 5 (filesystem, memory, sequential-thinking, chrome-devtools, sqlite) reported `reachable=false` with `installState=awaiting_pool_registration` and an `installHint` literally telling the operator: *"Register a managed pool to make this entry live: POST /api/pools..."* — but a fresh install ships them in that state with no operator action.

This violates `.claude/rules/00-mcos-realignment.md` *Old model to remove* bullet: *"Live-looking seeded infrastructure that is not actually configured, installed, reachable, and healthy."*

## 6. Risks and blockers

- **Persisted-pool drift.** The live MCOS service this session POSTed `memory` / `filesystem` / `sequential-thinking` / `chrome-devtools` pool definitions directly (the operational remediation). Those got persisted to `/api/config.pools`. On the post-v0.10.18 reboot the seed re-runs `upsertPool` for the same poolIds and the seed's canonical values win, overwriting the operator-customized config. This matches the existing v0.9.64 batch's behaviour and is documented in the seed comment.
- **Hardcoded nodejs path.** `C:\Program Files\nodejs\npx.cmd` is the standard Windows install location for nodejs LTS. Hosts that installed nodejs to a non-standard path will skip the seed and keep the placeholders. A future iteration could resolve npx via PATH lookup; the hardcoded path was the smallest patch and matches the project's existing pattern of hardcoded install paths.
- **Sqlite still placeholder.** Live testing showed sqlite would crash 5 times in 60 s and trigger the supervisor's 60 s quarantine cycle, producing 168 worker errors that surfaced on the operator's Diagnostic deck. Excluding it from the seed prevents the crash loop on fresh installs. Operators see a clean placeholder with an `installHint` instead.
