# v0.11.0-alpha.3 — Promotion Report

**Date:** 2026-07-02
**Type:** Version-authority promotion (repository cleanup branch `repo-cleanup/metadata-docs-ci`)
**Status:** Metadata promoted; MSI cut pending on the Windows host

## Why this release exists

The 2026-07-02 repository audit found version-authority drift: `VERSION.json` said `0.11.0-alpha.2`, `README.md` opened with alpha.1 language, the wiki claimed v0.10.11/v0.10.14, and 17+ source/test files carried `v0.11.0-alpha.3` implementation annotations with no canonical release metadata behind them. The alpha.3 functionality was already merged (PR #15, `fix/bug-campaign-2026-06`), so the audit's recommended resolution was promotion: declare `0.11.0-alpha.3` the single current version and synchronize every metadata surface to it.

## Scope (what the alpha.3 tree actually contains)

- **PHASE-14 completion, Slices B–E.**
  - Slice E: `SqliteDiagnosticsStore` (WAL journal, `schema_version` migration) + `DiagnosticsService` (1000-record ring + store, boot self-test persistence with last-50-boots retention, audited clear). `POST /api/diagnostics/clear` is functional (clear-with-retention), replacing the Slice A 501. Diagnostics routes are store-backed with jsonl fallback.
  - Slice B: six `mcos_diagnostics_*` tools in the `mcos-bridge` MCP plugin.
  - Slice C: WinUI Shell `DiagnosticsSectionControl` (severity/source filters, native save-dialog exports, ContentDialog-confirmed clear).
  - Slice D: browser dashboard Diagnostics tab with Blob-download exports.
- **Security hardening** (closes three alpha.2 deferrals): UDP beacon payload signing (HMAC-SHA256, key auto-generated on fresh installs; legacy configs broadcast unsigned), opt-in SChannel TLS for the admin HTTP listener (plain-HTTP fallback on credential failure; SSE over TLS returns 501 this iteration), weekly cert auto-rotation via `scripts/Register-CertAutoRotation.ps1`.
- **2026-06 bug campaign fixes:** beacon advertised the admin port as `gatewayPort`; exports composed around the retired port-7200 gateway URL; silent UDP beacon send failures; empty-`method` supervisor events.
- **Diagnostics log rotation** age (14-day) and count (200k-row) bounds alongside the v0.10.21 size bound.
- **Shell UX:** tile-grid expand-on-click endpoint detail.

## Remaining steps (Windows host)

1. Run the full validation chain: `cmake --preset debug && cmake --build --preset debug && ctest --preset debug --output-on-failure`, then the release preset equivalents.
2. Package: `scripts/Package-MasterControlOrchestrationServer.ps1 -Preset release -SkipBuild`.
3. Cut the GitHub Release `v0.11.0-alpha.3` through the gated release workflow (same-SHA product-gate rule applies).
4. Backfill the alpha.3 `history[0].commit` field (currently `pending`) with the cut commit.

Until step 3 completes, `v0.11.0-alpha.3` is the canonical repository version but has no published MSI artifact; `v0.11.0-alpha.2` remains the newest installable cut.
