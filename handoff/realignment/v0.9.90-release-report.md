# Release Completion Report — v0.9.90

Relative-time formatting for the supervisor heartbeat on the WinUI Shell — a deferred item from v0.9.82.

## Scope completed

### `formatRelativeUtcTime` helper + two card rewrites

Pre-v0.9.90 the WinUI Shell's Supervisor cards (Overview + Telemetry sidebar) rendered the heartbeat as a raw ISO-8601 UTC stamp:

```
Status: connected | Last heartbeat (UTC): 2026-05-11T00:57:59Z
```

The operator had to do mental arithmetic against "now" to figure out whether the supervisor was heart-beating recently or had gone wedged. v0.9.90 adds a `formatRelativeUtcTime(isoStampUtc)` helper in `ShellFormatting.h`:

* Parses `%Y-%m-%dT%H:%M:%SZ` via `std::get_time` + `_mkgmtime`.
* Computes delta = `now - stamp`.
* Buckets into operator-friendly units:
  * `<60s` → `"Xs ago"`
  * `<60m` → `"Xm Ys ago"` (or `"Xm ago"` when seconds == 0)
  * `<24h` → `"Xh Ym ago"` (or `"Xh ago"` when minutes == 0)
  * `>=24h` → `"Xd Yh ago"` (or `"Xd ago"` when hours == 0)
* Empty / parse-failure input returns empty wstring so callers fall back to raw UTC.
* Future timestamps (clock skew + freshly-stamped values) render as `"just now"` instead of `"-3s ago"`.

Both Supervisor card surfaces wired:

* **Overview** (`OverviewSectionControl::ApplySupervisorCard` connected branch): heartbeat phrase becomes `"| Last heartbeat: 5s ago"` instead of `"| Last heartbeat (UTC): 2026-05-11T00:57:59Z"`. Empty heartbeat renders `"| Last heartbeat: (none yet)"`. Parse failure falls back to raw UTC.
* **Telemetry sidebar** (`TelemetrySectionControl::ApplySnapshot` Supervisor block connected branch): same heartbeat phrase reformat.
