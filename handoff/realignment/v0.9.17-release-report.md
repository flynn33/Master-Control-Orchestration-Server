# Release Completion Report — v0.9.17

Maintenance release. Closes the v0.9.13-deferred persistence health surface for the activity ring.

## Scope completed

### `/api/activity/health` endpoint

Pre-v0.9.17 the activity ring's `appendToDiskLocked` had a `catch (...) {}` that silently swallowed every disk-write failure. Operators had no signal whether persistence was healthy. v0.9.17 adds:

- **Counters**: `persistedAppendCount_` (incremented on every append attempt), `persistedAppendErrorCount_` (incremented on each open / write failure).
- **Last-error capture**: `lastPersistErrorMessage_` (string with exception text or open/write detail), `lastPersistErrorAt_` (steady_clock timestamp).
- **Specific catch arms**: replaced the bare `catch (...) {}` with `catch (const std::exception& ex)` (captures `ex.what()`) and a fallback `catch (...)` with a "unknown" sentinel.
- **Public health snapshot**: `ActivityEventRing::persistenceHealth()` returns a `PersistenceHealth` struct under `mutex_`.
- **HTTP route**: `GET /api/activity/health` returns:

```json
{
  "persistenceEnabled": true,
  "filePath": "C:\\ProgramData\\MasterControlOrchestrationServer\\activity.jsonl",
  "bytesOnDisk": 45683,
  "totalAppends": 10,
  "totalAppendErrors": 0,
  "lastErrorMessage": "",
  "lastErrorSecondsAgo": -1,
  "inMemoryRingSize": 164,
  "highWaterMarkId": 164,
  "ok": true
}
```

`ok` is a top-level convenience flag (`persistenceEnabled` AND zero errors). Operators script `curl … | jq .ok` to gate alerts.
