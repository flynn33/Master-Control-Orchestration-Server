# Release Completion Report — v0.9.78

Closes two deferred items from the v0.9.76 / v0.9.77 train.

## Scope completed

### 1. Supervisor heartbeat watchdog

The supervisor lifecycle in v0.9.76 has six states: `off`, `config_generated`, `pending_connection`, `connected`, `disconnected`, `revoked`, `error`. v0.9.76 / v0.9.77 implemented every transition *except* `connected → disconnected` — there was no way for the server to notice that a remote supervisor client had stopped heart-beating. v0.9.78 closes that gap.

**Service-side method.** Added `ISupervisorAssignmentService::expireConnectionIfStale(threshold)` (header v0.9.78, implementation `SupervisorAssignmentServiceImpl::expireConnectionIfStale`). Parses `lastHeartbeatUtc` back into a `time_t` via `_mkgmtime`, compares against `now - threshold`, and transitions `Connected → Disconnected` when stale. No-op for any other state. Persists the new state to disk and returns `true` if a transition actually fired so the caller can audit-log.

**HTTP route.** Added `POST /api/supervisor/heartbeat`. Body is optional; carries `{"clientId":"..."}` so the runtime can ignore heartbeats from a stale supervisor that's been superseded by reassignment. Calls `recordHeartbeat(clientId)` on the service then returns the fresh `SupervisorStatus` as the response. Added to the route advertisement table.

**Runtime sweep.** `MasterControlApplication::Impl::runInteractive()` now ticks a 5s sweep that calls `expireConnectionIfStale(std::chrono::seconds(120))`. When a transition occurs, an `ActivityEvent{kind="supervisor_disconnected", actor="supervisor-watchdog", target=providerId, statusCode=504}` is appended to `globalActivityRing()` so the Overview Error Reporting card and `/api/activity` surface the lifecycle change. Threshold is generous (120s) so a single missed heartbeat from a connector with retry/backoff doesn't trigger spurious disconnects.

**Tests.** Two new backend tests in `MasterControlOrchestrationServerTests.cpp`:

* `testSupervisorHeartbeatWatchdogIdleStateNoTransition` — confirms the watchdog no-ops on `Off` and `ConfigGenerated` (only `Connected` is subject to the timeout).
* `testSupervisorHeartbeatWatchdogFlipsConnectedToDisconnected` — selects a Grok supervisor, confirms a fresh connection, sleeps 2s, calls `expireConnectionIfStale(std::chrono::seconds(1))` and verifies the state flips to `Disconnected`. Also verifies idempotency: re-running on a `Disconnected` assignment returns `false`.

Total Supervisor backend tests now: 14 (was 12 in v0.9.76).

### 2. `scripts/Deploy-LocalLive.ps1` helper

The v0.9.77 release report documented the root cause of the operator's "the server will not even start" report: `cmake --install` writes to `C:\Program Files\MasterControlOrchestrationServer\` (no spaces) but the Windows service binary path is `C:\Program Files\Master Control Orchestration Server\` (with spaces). The canonical deployment path is the WiX MSI installer, but during development hand-copying the new binaries every build is friction.

`scripts/Deploy-LocalLive.ps1` does the round-trip:

1. Stops the `MasterControlProgram` service.
2. Stops any running `MasterControlShell.exe` processes (so the shell binary + PDB aren't locked).
3. Copies `MasterControlServiceHost.exe`, `MasterControlShell.exe`, `MasterControlShell.pdb` (if present), `app.js`, `styles.css`, and `index.html` (if changed) from the build tree into the spaces-path live install.
4. Starts the service and polls `/api/version` until it responds (30s timeout).
5. Probes `/api/self-tests` and `/api/supervisor/status` as sanity checks.
6. If `-RelaunchShell` is set, launches `MasterControlShell.exe` and reports the PID + exit-status.

Idempotent. `$PSScriptRoot` fallback for both `&` and `-File` invocation styles. The script self-validated as part of the v0.9.78 deploy chain — used it to ship this release.

## Public contracts changed

* **NEW** route `POST /api/supervisor/heartbeat`. Body is the optional `{"clientId":"..."}`; response is the same `SupervisorStatus` shape `/api/supervisor/status` returns. Failure modes match the rest of the supervisor surface: 503 when the service hasn't initialized; 200 otherwise (recordHeartbeat is forgiving — if the state isn't Connected, the heartbeat is silently no-op'd).
* **NEW** activity-ring kind `supervisor_disconnected` (statusCode=504, actor="supervisor-watchdog"). Joins the v0.9.76 supervisor_* kinds.
* No breaking changes to existing public contracts.

## Risks and blockers

None. The watchdog has a 120s threshold so existing connected supervisors with normal retry backoff stay Connected; only a genuinely silent supervisor flips to Disconnected. The transition is reversible — the supervisor simply needs to call connect/confirm again to re-acquire.
