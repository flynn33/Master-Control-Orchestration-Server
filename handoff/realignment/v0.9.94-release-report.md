# Release Completion Report — v0.9.94

Future-direction relative-time formatter for the WinUI Shell.

## Scope completed

### `formatFutureUtcTime` helper + supervisor expiry rewrite

v0.9.90 added `formatRelativeUtcTime` for past timestamps (heartbeat → "5s ago"). The Overview Supervisor card's detail row (v0.9.79) was still rendering `expiresAtUtc` as a raw ISO-8601 stamp:

```
Expires (UTC): 2026-05-10T21:36:10Z
```

That requires mental arithmetic to know whether the config is about to expire. v0.9.94 adds the symmetric `formatFutureUtcTime` helper:

```cpp
inline std::wstring formatFutureUtcTime(const std::wstring& isoStampUtc);
//   future > now   → "in 47m 30s" / "in 2h 14m" / "in 3d 4h"
//   past   ≤ now   → "expired Xs ago"   (delegates to formatRelativeUtcTime)
//   parse fail     → ""                 (caller falls back to raw stamp)
```

The Overview Supervisor card detail row now reads:

```
Expires: in 2h 14m (2026-05-10T21:36:10Z)
```

Relative phrase up front, raw UTC stamp in parens for cross-check. Same dual-display pattern the dashboard's v0.9.74 wildcard-bind substitution uses (resolved value + raw value visible together).

The heartbeat line (added v0.9.79, reformatted v0.9.90) keeps its existing shape; only the expiry rendering changes in this release.
