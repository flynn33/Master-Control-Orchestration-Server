# Release Completion Report — v0.9.25

Maintenance release. Real operator-visible bug found in the bug-hunt.

## Scope completed

### Dashboard reachability now reflects supervised-pool truth, not the legacy TCP probe

The v0.9.25 bug-hunt found this: every mcp-server entry in the inventory has a legacy host:port (e.g. `192.168.1.7:7101` for `filesystem`, `7102` for `memory`, ...). Pre-v0.9.25 the dashboard's reachability dot was the result of a TCP-connect probe to that host:port. **But supervised pools spawn their workers as MCOS-managed stdio children, not as TCP listeners** — nothing's listening on `:7101` or `:7102` even when the `filesystem` and `memory` pools are alive and serving 14+9 tools through the gateway.

Operator-visible symptom: the dashboard showed red dots for `filesystem`, `memory`, `sequential-thinking`, `sqlite`, even while `tools/list` aggregated their tools and `tools/call` against them returned real data.

The fix: when a sub-agent / mcp-server has a supervised pool with at least one Ready instance, override `reachable=true` and replace the `endpointHostPort` display with `"stdio (supervised pool: N ready instance(s))"`. The truth from the supervisor wins over the meaningless TCP probe.

Inventory entries WITHOUT a supervised pool (the dormant 23-server catalog stubs — `client-tracker`, `metrics`, `computer-use`, `desktop-control`, `playwright`, `chrome-devtools`, `terminal-shell`, ... 23 entries with assigned but unimplemented host:ports) keep their TCP-probe-based reachability so the red dots there remain honest.

**Verified live**:

```
mcpServerId          poolId               reachable  endpointHostPort
-----------          ------               ---------  ----------------
filesystem           filesystem           True       stdio (supervised pool: 1 ready instance)
memory               memory               True       stdio (supervised pool: 1 ready instance)
sequential-thinking  sequential-thinking  True       stdio (supervised pool: 1 ready instance)
sqlite               sqlite               True       stdio (supervised pool: 1 ready instance)
```

(The dashboard view + browser dashboard's MCP-server card grid + the WinUI shell's MCP Servers deck all consume `mcpServerRuntimeStats`, so all three surfaces correct in lockstep.)
