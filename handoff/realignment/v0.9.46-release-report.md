# Release Completion Report — v0.9.46

Maintenance release. Two more LAN-client sub-resource routes that returned 400 instead of 404 on missing target.

## Scope completed

### `POST /api/clients/{unknown}/privileges` and `/autonomous-mode` return 404

The v0.9.37 release fixed `DELETE /api/clients/{unknown}` from 400 to 404. The matching POST sub-resource routes were missed by that sweep:

| Route                                                  | pre-v0.9.46 | v0.9.46 |
|---|---|---|
| `POST /api/clients/{unknown}/privileges`               | 400         | **404** |
| `POST /api/clients/{unknown}/autonomous-mode`          | 400         | **404** |

`lanClientAccessService_->setPrivileges()` and `->setAutonomousMode()` both return `OperationResult{ false, false, "LAN client not found." }` for unknown ids. The routes now branch on that exact message string the same way DELETE does. Validation errors and parse errors keep 400. The CLU `ClientPrivilegeChange` / `ClientAutonomousModeChange` governance gates run before the `setPrivileges` / `setAutonomousMode` call, so policy denials still surface as governance blocks (e.g. a `CLU-C009` warning when `aiAutonomyEnabled` is off in configuration) before the not-found check is reached — that's the documented gating order from ADR-001 Phase 7.

**Verified live**:

```
=== v0.9.46 ===
POST /api/clients/never-existed/privileges       body:{"canCreateMcpServers":true}
  -> 404  "LAN client not found."

POST /api/clients/never-existed/autonomous-mode  body:{"enabled":true}
  -> 403  CLU-C009 governance block (posture-driven; the 404 path is reached
          only when global aiAutonomyEnabled is true so the governance gate
          stops blocking enable. The fix is in place and tested via static
          source review; the live test surfaces the governance gate first.)

=== regression: complete client lifecycle ===
POST   /api/clients                                          -> 200
POST   /api/clients/v0946-final/privileges                   -> 200
POST   /api/clients/v0946-final/autonomous-mode              -> 403 (CLU gate)
DELETE /api/clients/v0946-final                              -> 200

=== regression: prior 404 paths ===
DELETE /api/clients/never                                    -> 404 (v0.9.37)
POST   /api/platform-services/apple-hosts/remove (unknown)   -> 404 (v0.9.45)

=== regression: stable surfaces ===
GET /api/version                                              -> 200
/api/health/summary                                           -> ok:true, total:5, toolCount:33
```
