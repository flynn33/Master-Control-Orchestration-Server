# Release Completion Report — v0.10.7

Operator-directed corrective release. The v0.10.6 reading of the Telemetry density directive was wrong:

* The reference visual is the **SUB-AGENT GRID cross-tab footer** at the bottom of the Overview deck — small bordered tiles, side by side, stacking vertically when the item count exceeds one row.
* v0.10.6 instead produced flat thin text rows AND hid the footer on Telemetry. Both were wrong.

v0.10.7 corrects both:

1. The compact path in `EndpointStatCardGrid.h` now renders the same tile shape as the SUB-AGENT GRID footer, laid out in a 7-column Grid that wraps to additional rows.
2. `MainWindow::ApplySubAgentFooter` no longer hides the footer on the Telemetry destination — the footer is the reference visual, so it stays visible.

## Scope completed

### Tile shape (matches SUB-AGENT GRID footer)

A new helper `endpoint_stat_card_grid_detail::buildFooterStyleTile<StatT>` builds one tile per stat with the exact visuals from `MainWindow::ApplySubAgentFooter` (v0.7.8):

| Property | Value |
|---|---|
| Border background | `#18FF3D2E` (faint TRON-red fill) |
| Border stroke | `#55FF3D2E` (TRON-red, 1px) |
| Corner radius | 6px |
| Padding | 8px horizontal, 6px vertical |
| Title | NAME (uppercase, 11pt semibold, cyan `#B8FFF0`) + 8x8 reachability dot |
| Specialization | 10pt neutral text (omitted if empty) |
| Util row | `XX%` label (10pt, MinWidth 28) + `ProgressBar` (h=4, MinWidth 60) + `A/B` ratio (10pt neutral) |
| Clients block | `no clients` if empty, else up to 2 lines of `ip (type)` + `+N more` |

The dot reachability semantics also match the footer: when `endpointHostPort` is empty the dot is neutral (we cannot probe an unconfigured endpoint); when configured, green if reachable, red otherwise.

### Grid layout

The compact branch builds a `Grid` with:

* 7 columns (each `1*` star width)
* `ceil(N / 7)` auto-height rows
* Tile `i` placed at `(row = i / 7, col = i % 7)`
* 10px ColumnSpacing and 10px RowSpacing

For this host's live counts:

* 26 MCP servers → 4 rows (7 + 7 + 7 + 5)
* 7 sub-agents → 1 row of 7

This matches the visual density of the existing SUB-AGENT GRID footer.

### Footer hide reverted

`MainWindow::ApplySubAgentFooter` no longer collapses `SubAgentFooterHeadline` / `SubAgentFooterGrid` when `currentDestination_ == kTelemetryDestination`. The footer renders on every destination, as it did before v0.10.6.

### Wide path untouched

`renderEndpointStatCardGrid(..., compact=false)` continues to produce the multi-line stacked cards used by `RuntimeSectionControl` and `OverviewSectionControl`. Nothing on those decks changed visually.

## Risks and blockers

* The tile grid uses a fixed 7-column count. On narrower window widths the tiles may compress to <100px wide and the util ProgressBar's MinWidth=60 will hit. If the operator wants the column count to track window width, that's a follow-up that would need ItemsRepeater + WrapLayout or a manual column-fit calculation; v0.10.7 keeps it simple to match the existing fixed-width footer pattern.

## Deferred work

* Per-tile pool note + per-tile endpoint host:port are intentionally not rendered in the tile grid (kept in the wide Runtime cards) to keep tile height ~80px. Operators who need that detail continue to use the Runtime tab.
* Phase 2 of the CLU rebuild and Phase 3 (enforceAction wiring) still ship in v1.0.0+.
* PHASE-14 DiagnosticsSectionControl with FileSavePicker export, pending operator approval gate.
* Telemetry log rotation still not implemented.
