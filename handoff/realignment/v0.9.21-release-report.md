# Release Completion Report — v0.9.21

Maintenance release. Two things: a new health-summary endpoint, and a real fix to the stop-gate hook from v0.9.18.

## Scope completed

### 1. `GET /api/health/summary` — single-URL operational view

Pre-v0.9.21 a monitoring tool / deploy gate / dashboard status chip had to fan out across:

- `/api/health` (ok+time)
- `/api/gateway/status` (state, mcpUrl)
- `/api/gateway/tools` (tool count)
- `/api/pools` (roster + per-pool instance state)
- `/api/activity/health` (persistence ring health)
- `/api/host/telemetry` (sampler liveness)

Six requests just to answer "is MCOS healthy?" v0.9.21 stitches them into one:

```json
{
  "version": "0.9.21",
  "time": "2026-05-09T20:02:48Z",
  "gateway": {
    "state": "running",
    "mcpUrl": "http://0.0.0.0:8080/mcp",
    "adapterType": "native",
    "toolCount": 33
  },
  "pools": {
    "total": 5,
    "withHealthyInstance": 5,
    "totalHealthyInstances": 5,
    "darkOrQuarantined": 0
  },
  "activityRing": {
    "enabled": true,
    "bytesOnDisk": 67824,
    "totalAppends": 10,
    "totalAppendErrors": 0
  },
  "hostTelemetry": {
    "sampleAvailable": true,
    "capturedAtUtc": "2026-05-09T20:02:49Z",
    "cpuPercent": 1.29,
    "memoryPercent": 35.0
  },
  "ok": true
}
```

The top-level `ok` boolean is the conjunction of every health signal: gateway state == running, every minInstances>0 pool has at least one healthy instance, zero pools dark/quarantined, persistence ring has zero errors, telemetry sampler has produced at least one reading. A monitoring tool can `curl … | jq .ok` to gate alerts.

**Verified live**: `ok:true` against the running 5-pool deployment.

### 2. Stop-gate hook moved into the loaded settings

The v0.9.18 stop-gate hook lived in `.claude/settings.local.json`, but that file isn't loaded into the active Claude Code session. The gate was inert. The third "are you stalled?" interruption this session pinned this down. v0.9.21 moves the hook into `.claude/settings.json` (the canonical loaded config), running alongside the existing echo-style reminder. The marker model (`.claude/iteration-mode.active` / `.claude/iteration-mode.disabled`) is unchanged; only the hook installation site moved.

This release also acknowledges that the stop-gate is a backstop — the primary mechanism is "keep iterating in the current turn." Three iterations have shipped between this stall and the previous one (v0.9.18, v0.9.19, v0.9.20, v0.9.21) which is at least progress in the right direction.
