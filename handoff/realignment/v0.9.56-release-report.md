# Release Completion Report — v0.9.56

Diagnostic-surface expansion. Unblocks the long-standing operator complaint that the dashboard reported "X of Y reachable" with no explanation for the unreachable entries and no way to tell apart the three classes of catalog entry MCOS actually has.

## Scope completed

### Honest-unavailable diagnostic fields on every runtime stat

`SubAgentRuntimeStat` and `McpServerRuntimeStat` each gained five new fields:

| Field | Type | Purpose |
|---|---|---|
| `unavailableReason` | string | Categorized failure: `connection_refused` / `timeout` / `dns_failed` / `host_unreachable` / `socket_error` / `no_listener` / `supervised_pool_not_ready` / `no_endpoint_registered`. Empty when reachable. |
| `lastErrorMessage` | string | Human-readable line including `host:port`, e.g. `"TCP connect to 192.168.1.7:7104 timed out after 200ms - target may be firewalled or down."` |
| `lastErrorAtUtc` | string | ISO-8601 UTC stamp of the last failed probe. Empty for healthy entries. |
| `installState` | string | `installed_and_supervised` / `online_via_admin_port` / `online_via_external_listener` / `supervised_pool_not_ready` / `awaiting_pool_registration` / `unknown`. |
| `installHint` | string | Actionable next step, e.g. `"Catalog placeholder for MCP server 'computer-use'. No process listens on 192.168.1.7:7104. Register a managed pool to make this entry live: POST /api/pools with poolId=\"computer-use\" and an executable template."` |

The fields appear in `/api/dashboard` (`mcpServerRuntimeStats[]` + `subAgentRuntimeStats[]`) and the new `/api/diagnostics/runtime-stats` endpoint.

### Probe failure category captured in the reachability cache

`ReachabilityCacheEntry` now stores `lastErrorCategory`, `lastErrorMessage`, and `lastErrorAtUtc` alongside the boolean `reachable`. The Win32 probe path discriminates `getaddrinfo` failure (DNS), `WSAECONNREFUSED` (no listener bound), `WSAEHOSTUNREACH` / `WSAENETUNREACH` (routing failure), `select` timeout vs other socket errors, and synthesizes a stable category + human-readable message for each. The snapshot pipeline reads back the cache via `lastReachabilityDiagnostic(hostPort)` and mirrors the values into the per-entry diagnostic fields.

### Snapshot pipeline classifies install state for every entry

A new private template helper `enrichRuntimeStatDiagnostics<Stat>(...)` runs at the end of both the sub-agent and MCP-server snapshot loops and classifies each entry into one of six install states:

* **`installed_and_supervised`** — managed stdio pool wraps the entry and has ≥1 Ready instance. Hint: pool id + ready count.
* **`supervised_pool_not_ready`** — pool registered but 0 Ready instances. Hint: inspect supervisor / Job Object exit codes.
* **`online_via_admin_port`** — reachable at the MCOS admin port (e.g., the three governance MCP entries). Hint: register a pool for supervised lifecycle.
* **`online_via_external_listener`** — reachable at an external `host:port` but not under MCOS supervision. Hint: register a pool.
* **`awaiting_pool_registration`** — TCP probe failed AND no pool exists. The catalog placeholder case (21 of 28 MCP servers + all 7 sub-agents on this host).
* **`unknown`** — fallback; should be unreachable in practice.

### New `GET /api/diagnostics/runtime-stats`

Bundles per-entry detail with an aggregate summary so operators answer "how many of my catalog entries are actually live?" in one request. Response shape:

```json
{
  "version": "0.9.56",
  "generatedAtUtc": "...",
  "mcpServers": {
    "summary": {
      "total": 28, "reachable": 7, "unreachable": 21,

```

## Risks and blockers

**No breaking changes.** Field-additive; pre-v0.9.56 clients ignore the new fields. The new endpoint is opt-in.

**Cache-key alignment risk:** `enrichRuntimeStatDiagnostics` looks up the cache by formatting `host:port` from the seeded endpoint, NOT from `stat.endpointHostPort` (which v0.9.25 truth-wins rewrites to "stdio (supervised pool: ...)"). Validated against live data: the cache lookup hits and produces correct `unavailableReason=timeout` for placeholders.

**Probe-category accuracy:** Windows non-blocking connect commonly returns timeout (no RST within 200ms) rather than `WSAECONNREFUSED` for unbound local TCP ports — observed for all 21 MCP placeholders + 7 sub-agent placeholders. Both categories ("timeout" and "connection_refused") are surfaced honestly; operators just get the underlying Win32 truth, not a fabricated label.
