# Release Completion Report — v0.9.22

Maintenance release. Surfaces v0.9.21's `/api/health/summary` URL in the discovery admin block.

## Scope completed

`DiscoveryAdmin` gained `healthSummaryUrl` so monitoring tools and deploy gates can find the health-summary endpoint via `/.well-known/mcos.json` or `/api/discovery` instead of hard-coding the path. Same pattern as the v0.9.18 `activityHealthUrl` addition.

```
discovery.admin.healthSummaryUrl = http://192.168.1.7:7300/api/health/summary
GET that URL → { ok: true, ... }
```

Verified live: discovery returns the URL, fetching it via the discovered URL returns `ok:true`.

The v0.9.22 bug-hunt also stress-probed the health-summary endpoint with concurrent worker kills: the summary call returned in 7ms during a kill, and `ok` recovered to `true` 5s later (the watchdog reaped + auto-respawned in that window).
