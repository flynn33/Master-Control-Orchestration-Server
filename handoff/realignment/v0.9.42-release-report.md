# Release Completion Report — v0.9.42

Maintenance release. Closes the 500-on-bad-body sweep across the remaining six POST routes.

## Scope completed

### Six more POST routes return 400 instead of 500 on bad bodies

The v0.9.42 bug-hunt swept every POST route and found six more sites still returning 500 on empty / malformed bodies — the same pattern v0.9.38 (runtime registration) and v0.9.41 (install routes) closed. Each helper does `nlohmann::json::parse(requestBody).get<Spec>()` inline; the parse_error throws past the route's local 200/400 branch and hits the outer 500 handler. Wrapping in try/catch fixes each.

| Route                                              | Service helper                       | pre-v0.9.42 | v0.9.42 |
|---|---|---|---|
| `POST /api/config`                                 | `applyConfigurationJson`             | 500         | **400** |
| `POST /api/platform-services/apple-hosts`          | `upsertAppleRemoteHostJson`          | 500         | **400** |
| `POST /api/runtime/subagent-groups`                | `upsertSubAgentGroupJson`            | 500         | **400** |
| `POST /api/forsetti/modules/state` (malformed)     | inline `nlohmann::json::parse`       | 500         | **400** |
| `POST /api/clu/execute`                            | `executeGovernanceToolJson`          | 500         | **400** |
| `POST /api/clu/apple-operations/cancel`            | `cancelAppleOperationJson`           | 500         | **400** |

`/api/forsetti/modules/state` was already special-cased to `{}` for an empty body; the fix only applies to *malformed* JSON (e.g. `not-json`).
