# Release Completion Report — v0.9.86

Pure refactor — pays down the duplication v0.9.85's release report flagged.

## Scope completed

### Hoist wildcard-bind substitution helper

v0.9.84 added an inline `resolveGatewayUrl` lambda in `/api/health/summary`. v0.9.85 added a parallel `resolveWildcardUrlV2` lambda in `/api/gateway/status` + `/api/gateway/health` (slightly extended to substitute the URL inside narrative-text positions too). Three nearly-identical copies, drifting slightly already, all doing the same substitution rule.

v0.9.86 hoists the work into a single free helper at the top of `MasterControlRuntime.cpp`, alongside the existing `resolveMcosServerHost`:

```cpp
std::string substituteWildcardHostInUrl(const std::string& raw,
                                          const AppConfiguration& configuration);
```

All three callers now call it directly with `(field, configurationService_->current())`. Helper handles both URL-prefix matches (`://0.0.0.0:8080/mcp`) and free-text matches (`Native HTTP.sys gateway listening on http://0.0.0.0:8080/...`).

Behavior is byte-identical to v0.9.85. The release smoke confirms: `/api/gateway/status` still returns `mcpUrl=http://192.168.1.7:8080/mcp` (substituted) + `mcpUrlRaw=http://0.0.0.0:8080/mcp` (original), no change in response shape.

This unblocks future callers that need the same substitution — they can call the helper directly instead of carrying yet another inline lambda copy.
