# Release Completion Report — v0.9.99

Three operator-named issues fixed after the v0.9.98 stale-XAML fix unblocked actual visibility on the live shell.

## Operator report

> Where is the visible self test at start up?
> Where is the error window on "overview"
> The MCP cards are not the size I asked for

All three are real complaints about the *current* WinUI Shell surface. Pre-v0.9.99 the Self-Tests + Error Reporting cards existed but were buried below five other sections on the Overview tab, and the "compact" MCP cards I shipped in v0.9.76 only shrunk fonts/padding — the cards still emitted 5-6 rows each.

## Scope completed

### 1. Self-Tests + Error Reporting promoted to top of Overview

Pre-v0.9.99 Overview layout, top to bottom:
1. Hero header
2. Claude Code Control toggle
3. Operational Snapshot narrative
4. Environment + Configuration Narratives (2-column)
5. APIs / Security / MCP / Sub-Agents status grid (2x2)
6. **Self-Tests card** ← position 6
7. Supervisor Agent card
8. **Error Reporting card** ← position 8

Operator's directive on this surface was *"MCOS MUST perform visible self tests... If a test fails, then it MUST be reported in the Overview Error Frame."* "Visible" means visible on startup, not "exists five scroll-pages down."

v0.9.99 layout, top to bottom:
1. Hero header
2. **Self-Tests + Error Reporting** — 2-column row directly under the hero, both cards side-by-side, no scroll required ← NEW POSITION
3. Claude Code Control toggle
4. Operational Snapshot
5. Environment + Configuration
6. APIs / Security / MCP / Sub-Agents status grid
7. Supervisor Agent card

Both cards keep the same `x:Name` elements so the existing `ApplySelfTestCard` / `ApplyErrorReportingCard` C++ handlers populate them without any code change.

### 2. MCP / Sub-Agent cards on Telemetry are now actually compact

The original operator directive was *"The MCP Servers on the telemetry page MUST have smaller cards. They MUST be close in size to the current Sub-Agent Cards."*

What v0.9.76 shipped: a `compact` boolean on `renderEndpointStatCardGrid` that tightened padding (14→10) and fonts (16→13) but kept the multi-row card structure (header + util row + endpoint row + pool note + clients header + clients list). Result: cards were ~140 pixels tall instead of ~180. Still bulky; with 28 MCP servers on the Telemetry deck that's ~4000 vertical pixels of cards.

What v0.9.99 ships: compact is now a TRUE single-line card. Each card is one horizontal `Grid` row with five cells:

```
[dot] Name (spec)            [████░░░░░░░] util%   active/cap
```

* **dot** — 10×10 reachability indicator (green/red)
* **Name (spec)** — horizontal stack with `CharacterEllipsis` trimming on overflow
* **util bar** — slim `ProgressBar`, tone-colored (good/warn/crit)
* **util%** — small bold percentage in the tone color
* **active/cap** — `N / M` counter on the right

~28 pixels tall per card. 35 cards now fit in ~1000 pixels on the Telemetry deck. Same template used for both MCP-server and sub-agent decks so they're visually identical on Telemetry, which is what the operator asked for ("close in size to the current Sub-Agent Cards").

The Runtime tab and the Overview surface cards (which use `compact=false`) still render the full multi-row form for primary inventory work where the operator needs to see endpoint host:port + pool stats + active-client list.

## Risks and blockers

The compact single-line layout drops the per-card endpoint host:port, pool stats, and active-client list. Those signals still exist on the Runtime tab and via `/api/dashboard`. If operators want them on the Telemetry deck (expand-on-click, popover, etc.), that's a v1.0.0+ candidate.
