# Release Completion Report — v0.9.70

Telemetry deck now surfaces the MCP server card grid alongside the existing sub-agent card grid. Per operator directive, each MCP server gets its own small card with the same visual structure as a sub-agent card: reachability dot, utilization bar with capacity counts, supervisor pool name, last-probed timestamp, install-state pill + diagnostic block (v0.9.56-58), connected-client list (active leases through the gateway). Reads `state.dashboard.mcpServerRuntimeStats` which the runtime already populates every snapshot tick.
