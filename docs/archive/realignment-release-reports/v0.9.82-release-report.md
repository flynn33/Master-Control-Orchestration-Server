# Release Completion Report — v0.9.82

Small correctness fix on the v0.9.79 tray "Open Browser Dashboard" entry.

## Scope completed

### Tray Open Browser respects live browser port

The v0.9.79 tray entry called `ShellExecuteW(... "open", L"http://127.0.0.1:7300/", ...)`. That URL is correct for the default deployment but silently breaks whenever the operator has configured a non-default browser port via Settings, or has changed the bind address. v0.9.82 reads `self->currentSnapshot_.dashboardUrl` instead — which is constructed by `ShellRuntime::CaptureSnapshot` with the wildcard substitution + LAN-routable IP resolution + the live browser port already applied. Two fallbacks keep the entry resilient:

1. If `dashboardUrl` is empty (very early shell startup before the first snapshot lands), fall back to `http://127.0.0.1:<live-browser-port>/`.
2. If `browserPort` is also 0 (impossible after `ConfigurationService::current()` populates it), fall back to the legacy `http://127.0.0.1:7300/` so the tray entry never produces an empty URL.

The fix is one switch-case body inside `MainWindow::TrayHostWndProc`. The tray menu order, command IDs, and other entries are unchanged.
