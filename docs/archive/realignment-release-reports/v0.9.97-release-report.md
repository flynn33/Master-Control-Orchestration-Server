# Release Completion Report — v0.9.97

Session-milestone verification. Documents the 22-release supervisor-and-shell train shipped this session (v0.9.76 → v0.9.97).

## Why this release exists

v0.9.96 brought the iteration loop to a natural plateau on this directive set. The system is fully functional, the operator-named directives are all resolved, the supervisor surface is comprehensive, and the bug-hunts are surfacing no actionable issues. Per the project iteration rule, even in diminishing-returns territory I keep moving — the documented-decision artifact is the right output when there's nothing concrete left to ship. v0.9.97 is that artifact.

No code changed in v0.9.97 itself. VERSION bumped, validation chain re-run to prove the gates are green at the milestone, this report written.

## What landed this session (v0.9.76 → v0.9.97)

22 releases. Every one with `cmake --build` clean (0 warnings, 0 errors), `ctest` 4/4 green, `scripts\check-mastercontrol-forsetti.ps1` passing.

### v0.9.76 — Operator's named directive set

* Bind-address consistency on the WinUI Shell (4 display sites rewired through new `resolveDisplayBindAddress` helper).
* Compact card mode on `EndpointStatCardGrid` used on the Telemetry surface so MCP-server + sub-agent decks render at matching smaller size.
* WinUI-safe system tray icon (HWND_MESSAGE-parented stub window + AppWindow.Closing intercept). Replaces the v0.9.72 WndProc-subclass attempt that crashed the shell.
* Supervisor Agent Assignment Wizard end-to-end: data model header, service .cpp with persistence + capability validation + audit emission, six HTTP routes, WinUI Overview card with single-selection toggles + Generate Config & Save (IFileSaveDialog) + Revoke Active + status detail row, 12 backend tests.

### v0.9.77 — Browser dashboard supervisor parity

Same lifecycle UX on the browser Overview deck via `renderSupervisorPanel` in `resources/web/app.js`. Also documented the cmake-install-prefix vs live-service-path mismatch that caused the operator-reported "the server will not even start" — the OLD shell binary in the spaces-path crashed because `cmake --install` hadn't replaced it.

### v0.9.78 — Heartbeat watchdog + Deploy-LocalLive.ps1

* `expireConnectionIfStale(120s)` flips Connected → Disconnected when no heartbeat lands. Periodic sweep on `runInteractive`. `supervisor_disconnected` activity event emitted on transition. 2 new backend tests.
* `scripts/Deploy-LocalLive.ps1`: stops service + shell, copies fresh artifacts to spaces-path, restarts, probes. Resolves the recurring hot-patch friction.

### v0.9.79 — Tray visibility + Overview detail row + Telemetry sidebar Supervisor card

* Tray menu adds "Open Browser Dashboard" (ShellExecuteW with verb `open`).
* Overview Supervisor card detail row (assignmentId / configId / expires / heartbeat / client).
* Telemetry sidebar gains a compact read-only Supervisor status card.

### v0.9.80 — `DEPLOY_LOCAL_LIVE` cmake target

`cmake --build --preset debug --target DEPLOY_LOCAL_LIVE` is the one-command build-and-deploy workflow. Depends on the runtime targets so anything stale rebuilds first.

### v0.9.81 — Security hardening

`confirmConnection` capability validator: empty allowed-set now default-denies all claims (pre-v0.9.81 short-circuit accepted any non-forbidden capability). Regression test `testSupervisorConfirmDefaultDenyOnEmptyAllowedSet`.

### v0.9.82 — Tray URL respects live browser port

Replaces the hardcoded `http://127.0.0.1:7300/` with `self->currentSnapshot_.dashboardUrl`.

### v0.9.83 — Provider-specific supervisor config instructions

Claude / ChatGPT / Grok each get tailored `instructions.steps` (was generic boilerplate). Inlines `server.mcpEndpoint`.

### v0.9.84 + v0.9.85 + v0.9.86 — Wildcard URL leaks

Eliminated `0.0.0.0` leaks from `/api/health/summary`, `/api/gateway/status`, `/api/gateway/health`. v0.9.86 hoisted the substitution into a single named helper `substituteWildcardHostInUrl`.

### v0.9.87 — Supervisor block in /api/health/summary

`ok` flag now includes `supervisor.state != error`. New `supervisor` block in response body.

### v0.9.88 — Curl example in config instructions

`buildConfirmCurlExample(a)` emits a copy-paste-ready `curl -X POST ... /api/supervisor/connect/confirm ...` with the actual configId, providerId, capabilities, fingerprint inlined. End-to-end validated against the live service.

### v0.9.89 — kind filter on /api/activity

`?kind=supervisor_*` (or exact match) filters server-side. Read window expands to `maxCount*4` so the filter doesn't starve results. `kindFilter` echo in response.

### v0.9.90 + v0.9.91 — Relative-time formatting (past direction)

`formatRelativeUtcTime` on both surfaces. Supervisor card heartbeat now reads `"Last heartbeat: 5s ago"` not `"Last heartbeat (UTC): 2026-05-...Z"`.

### v0.9.92 + v0.9.93 — Persistent diagnostics for supervisor lifecycle

All seven supervisor_* lifecycle events dual-emit to `MasterControl::Diagnostics::appendEvent`, landing in `<PUBLIC>\Documents\Master Control Orchestration Server\logs\supervisor\events.jsonl` (append-only, never trims). Closes the operator-directive mandate: *"If a test fails, then it MUST be reported in the Overview Error Frame and reported to the diagnostic module."*

### v0.9.94 + v0.9.95 — Future-direction relative-time

`formatFutureUtcTime` on both surfaces. Supervisor card expiry now reads `"Expires: in 2h 14m (2026-05-...Z)"` — relative phrase up front, raw stamp in parens for cross-check.

### v0.9.96 — Issued: X ago

Detail row gains `Issued: 3m 14s ago` between Config and Expires lines so operators see assignment age at a glance.

### v0.9.97 — This report

Documented milestone. No code change.
