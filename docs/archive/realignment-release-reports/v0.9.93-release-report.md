# Release Completion Report — v0.9.93

Completeness pass on the v0.9.92 persistent-diagnostics dual-emit.

## Scope completed

v0.9.92 wired five supervisor lifecycle events to `MasterControl::Diagnostics::appendEvent`. Two more cousin events fire on the `/api/supervisor/config/generate` route — they were left unmirrored in v0.9.92 because the route serves two distinct paths (provided-providerId → `supervisor_config_issue`; empty-body → `supervisor_config_regenerate`) and I missed both branches. v0.9.93 closes that gap.

Now the full set of supervisor lifecycle events fires both the in-memory activity ring `append` and the persistent `Diagnostics::appendEvent`:

| Event kind | v0.9.92 | v0.9.93 |
|---|---|---|
| `supervisor_select` | ✓ | ✓ |
| `supervisor_revoke` | ✓ | ✓ |
| `supervisor_connect` | ✓ | ✓ |
| `supervisor_connect_rejected` | ✓ | ✓ |
| `supervisor_disconnected` | ✓ | ✓ |
| `supervisor_config_issue` | — | **NEW** |
| `supervisor_config_regenerate` | — | **NEW** |

Both new dual-emits use severity=`info` on ok, `error` on rejection. Data block includes `providerId`, `assignmentId`, `configId`, `actor`.
