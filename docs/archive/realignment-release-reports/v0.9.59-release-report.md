# Release Completion Report — v0.9.59

At-a-glance summary banner above the MCP-server and sub-agent card grids. Pre-v0.9.59 the operator landing on the dashboard saw "MCP Servers (28)" in the heading and had to scan all 28 cards to count how many were live. v0.9.59 inserts a compact banner directly under each grid header that reads the same `mcpServerRuntimeStats` / `subAgentRuntimeStats` arrays already in `state.dashboard` and renders the reachable count + `installState` bucket pill row in one strip.

## Scope completed

### `renderRuntimeStatSummaryBanner(stats, kindLabel)` helper

Computes the same summary the v0.9.56 `/api/diagnostics/runtime-stats` aggregate produces, but client-side from data already in the dashboard payload — no new HTTP round-trip. Renders three rows:

1. **Headline** — `<reachable> / <total> reachable · <unreachable> <kindLabel>s need attention`. The reachable count is good (green) when all entries are reachable, warn (amber) when some are reachable, critical (red) when none are.
2. **Bucket pill row** — one pill per non-zero `installState` bucket: `supervised`, `online (admin port)`, `online (external)`, `pool stuck`, `no pool registered`, `unclassified`. Zero-count buckets are suppressed so a clean install shows only a single green "supervised: N" pill.
3. **Install-command tail** — *"<N> of the placeholders carry a known canonical install command — copy it from the entry's diagnostic block."* Only renders when at least one entry has a non-empty `installCommand` (v0.9.58 field).

### Wired into both grid renderers

`renderSubAgentUtilizationPanel` and `renderMcpServerUtilizationPanel` insert the banner between the existing header copy and the card grid `<div>`. The banner is suppressed when the stats array is empty (no double-empty-state).

### CSS chrome matches existing panel-block aesthetic

`.subagent-summary-banner` uses the same dark-glass linear gradient + cyan border as `.subagent-card` so it reads as part of the same control surface. The pill palette mirrors the per-card diagnostic pill palette (good/warn/critical/info → the established `#1cf2c1` / `#ffaf3a` / `#ff3a5a` / neutral gray).

## Risks and blockers

* **Banner is read-only.** The pills don't yet act as filter chips — clicking "no pool registered: 21" doesn't filter the card grid. That's a follow-up; out of scope for this iteration.
* **Two grids on the same panel double the banner footprint.** On the catalog deck both grids render banners back-to-back (~120px combined). Acceptable density given the value of the at-a-glance summary; revisit if operator feedback says the deck feels banner-heavy.
