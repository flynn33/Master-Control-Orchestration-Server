# Release Completion Report — v0.9.61

First in-tree backend that converts a previously-placeholder catalog id into a live, gateway-aggregated MCP server. Pre-v0.9.61 the catalog advertised `terminal-shell` at hardcoded port `7108` with nothing actually listening — it appeared in the dashboard as `awaiting_pool_registration` with `unavailableReason=timeout`, and the gateway's `tools/list` aggregate could not surface any `shell.*` tool. v0.9.61 (a) extends `mcos-baseline-tools-worker.exe` to dispatch by `--specialization=<id>`, (b) adds a `TerminalShell` specialization implementing `shell.exec(command, [cwd], [timeoutMs])` via `CreateProcessW` + anonymous pipes, and (c) auto-registers the supervised pool in `MasterControlRuntime` so a fresh boot exposes `terminal-shell__shell.exec` at the gateway alongside `filesystem__read_file` etc.

## Scope completed

### Multi-specialization worker

`mcos-baseline-tools-worker` accepts `--specialization=<id>` from argv at startup. Default `baseline-tools` preserves the v0.9.1 contract (mcos.echo / mcos.now / mcos.host_info / mcos.add). The `Specialization` enum branches `toolCatalog()` and `dispatchToolCall()`. The `serverInfo.name` in the `initialize` response now reflects the active specialization (`"MCOS terminal-shell Worker"`) so operator-side logs can tell pools apart.

### `runProcessCaptured` Win32 helper

CreateProcessW + anonymous pipes for stdout/stderr + alternating PeekNamedPipe / ReadFile drain + WaitForSingleObject + GetExitCodeProcess. Honors a configurable timeout (TerminateProcess on expiry, 1 MB stdout/stderr cap). Captures launch errors honestly without faking exit codes — `launchFailed=true` + `launchError="CreateProcessW failed (Win32 error <code>)"` bubbles back to the operator.

### `shell.exec` tool

Wraps the user command in `cmd.exe /S /C "<command>"` so pipes / redirection / built-ins (dir, set, echo) work. Default 30s timeout; min 100ms, max 5 min. Returns a JSON body inside the MCP text-content shape with `command`, `exitCode`, `timedOut`, `stdout`, `stderr`, `timeoutMs`, optional `cwd`, and `launchFailed`/`launchError` when CreateProcessW fails.

### Auto-registered `terminal-shell` pool

`MasterControlRuntime` upserts the pool at boot using the same baseline-tools-worker.exe binary with `args=["--specialization=terminal-shell"]`. minInstances=1 so it spawns immediately; maxInstances=2; maxActiveLeasesPerInstance=8.

## Architecture established

Pattern is now reusable for the remaining 20 placeholders: each adds a `Specialization` enum value, a catalog function, a dispatcher branch, and a pool registration. The same `mcos-baseline-tools-worker.exe` binary serves all specializations — one process per logical MCP, one binary on disk.
