# Release Completion Report — v0.9.79

Three WinUI Shell visibility additions on top of the supervisor surface (v0.9.76) + heartbeat watchdog (v0.9.78). Patch-level release — no public API change, no breaking change.

## Scope completed

### 1. Tray menu gains "Open Browser Dashboard"

The right-click tray menu shipped in v0.9.76 read: *Show MCOS Shell* / *Stop Server* / *Restart Server* / *Close MCOS Shell*. v0.9.79 adds **Open Browser Dashboard** between *Show* and the separator. ShellExecuteW with verb `open` resolves the operator's default browser without hardcoding Edge/Chrome/Firefox; the URL is the loopback `http://127.0.0.1:7300/` which the running service is listening on.

For remote-desktop operators (memory: "operator runs RDP into the host"), this is one-click access to the browser dashboard without touching the start menu or tabbing through windows.

### 2. Overview Supervisor card detail row

The v0.9.76 Supervisor card showed headline + status text. v0.9.79 adds a collapsed-by-default detail row (`SupervisorDetailText`, monospace font) that becomes visible whenever the assignment is active. It carries:

* `Assignment: SUP-{compactUtcStamp}-{hex}`
* `Config: CFG-{compactUtcStamp}-{hex}`
* `Expires (UTC): 2026-...`
* `Last heartbeat (UTC): 2026-...` (or `(none yet)` for a freshly-Connected assignment that hasn't heartbeated)
* `Client: {clientId}`

Hidden when `state=off` so the unconfigured card stays clean. Useful operator-facing debug info when something goes wrong — e.g., if the supervisor client reports the wrong configId or an expired token, the operator can read the live assignment / config / expiry fields directly without curling `/api/supervisor/assignment`.

### 3. Compact Supervisor status card on the Telemetry sidebar

The Telemetry surface previously only showed control-plane posture, runtime routes, telemetry narrative, environment narrative on its sidebar. v0.9.79 adds a fifth card: **SUPERVISOR AGENT** with status dot + headline + one detail line. Read-only — the toggle group + Generate / Revoke buttons stay on the Overview surface per the WinUI-is-priority directive.

Tone mapping mirrors the Overview card:
* `connected` → good (green dot, `Active: ChatGPT | Status: connected | Last heartbeat (UTC): ...`)
* `pending_connection` / `config_generated` → warn (orange dot, `... pending connection | Config expires (UTC): ...`)
* `disconnected` → warn (orange dot, `... disconnected | Heartbeat timeout. Remote client may have stopped.`)
* `revoked` → warn (`... revoked | Re-select a provider on the Overview deck to reassign.`)
* `error` → crit (red dot, error message inline)
* `off` → neutral (grey dot, `No supervisor assigned.`)

Operators monitoring telemetry now see supervisor lifecycle changes without needing to pivot to Overview.
