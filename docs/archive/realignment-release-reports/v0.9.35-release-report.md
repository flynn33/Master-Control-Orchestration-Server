# Release Completion Report — v0.9.35

Maintenance release. Governance route returning the wrong bundle on unknown platforms.

## Scope completed

### `/api/governance/bundles/{unknown-platform}` now returns 404, not the windows bundle

The v0.9.34 bug-hunt found that `/api/governance/bundles/freebsd` returned `200 OK` with a 10 417-byte body — and that body was byte-for-byte the same `windows` bundle (same SHA-256 checksum). Root cause in [`MasterControlRuntime.cpp:9670`](../../../src/MasterControlApp/MasterControlRuntime.cpp#L9670):

```cpp
static std::string normalizePlatform(const std::string& platform) {
    /* ... lowering + recognized variants ... */
    return "windows";   // every unknown input falls into Windows
}
```

`GovernanceBundleService::bundleFor()` then composed a windows bundle and shipped it. A misconfigured client requesting bundles for a platform MCOS doesn't actually support would *silently* get the wrong rules — and likely act on them, since the bundle has a valid checksum and the rest of the contract.

The fix validates the platform at the route layer with `platformFromKey()` (which already returns `PlatformTarget::Unknown` for unrecognized values) and returns 404 + a JSON body listing the supported platforms:

```json
{ "succeeded": false,
  "message": "Unknown governance bundle platform 'freebsd'. Supported: windows, macos, ios." }
```

The internal `normalizePlatform` fall-through is left in place — it's defense-in-depth for any future caller that bypasses the HTTP route (e.g. an internal CLU enforce-action path that constructs a bundle reference from a config file). The route is now the source of truth for what's externally addressable.

```
freebsd body:

  "requiresConfirmation": false,
  "succeeded": false

```

`osx` and `iphoneos` were previously accepted by `normalizePlatform` but were never documented or surfaced — they predate macOS naming (`macOS` since 2016) and are Apple SDK internal terminology. Returning 404 for them is a more correct response than coercing to `macos`/`ios`; the error body lists the canonical names.
