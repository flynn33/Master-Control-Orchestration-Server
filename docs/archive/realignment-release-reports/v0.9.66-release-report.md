# Release Completion Report — v0.9.66

Win32-input + screen-capture + desktop-control batch. Four new specializations. Reachable 20/28 → 24/28; gateway tools 57 → 71.

## Scope completed

* `Specialization::KeyboardMouseControl` / `ScreenCaptureVision` / `DesktopControl` / `ComputerUse` + CLI parser branches.
* **keyboard-mouse-control**: `input.keyboard` (`SendInput KEYEVENTF_UNICODE`, special-cased `\n`/`\t`/`\b` → VK_RETURN/VK_TAB/VK_BACK), `input.click` (`SetCursorPos` + MOUSEEVENTF_*), `input.move`, `input.scroll` (MOUSEEVENTF_WHEEL/HWHEEL).
* **screen-capture-vision**: `screen.capture` builds a 24-bit DIB via `GetDC` + `CreateCompatibleBitmap` + `BitBlt` + `GetDIBits`, prepends `BITMAPFILEHEADER` + INFO header so the byte stream is a valid `.bmp`, base64-encodes via the new `base64Encode` helper, returns under MCP image content shape with width/height metadata. `screen.size` via `GetSystemMetrics`.
* **desktop-control**: `desktop.list_windows` (`EnumWindows` callback collecting hwnd / title / visibility / foreground / bounds), `desktop.focus` (case-insensitive substring match + `SetForegroundWindow`), `desktop.launch` (`ShellExecuteW`; pulled in `<shellapi.h>` + `Shell32.lib`), `desktop.foreground` (`GetForegroundWindow` + `GetWindowTextW` + `GetWindowThreadProcessId`).
* **computer-use** umbrella re-exports the most common tools as `computer.screenshot` / `computer.click` / `computer.type` / `computer.move_mouse` / `computer.window_list` so clients written against the Anthropic computer-use spec can use MCOS as a drop-in.
* `MasterControlRuntime` upserts 4 supervised pools.

## Risks

* Service runs in session 0 → `screen.capture` and `desktop.list_windows` see the service desktop, not the operator's interactive session. For interactive use the worker would need to be relaunched in a user session via `CreateProcessAsUserW`. Documented; out of scope for this iteration.
