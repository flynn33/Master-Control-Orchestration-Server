# Release Completion Report — v0.9.71

Real-time push channel via Server-Sent Events. Pre-v0.9.71 the dashboard polled `/api/dashboard` every 2s; every refresh paid a full snapshot round-trip. v0.9.71 adds a streaming endpoint at `GET /api/events` that emits `dashboard` events whenever the snapshot changes (~1s tick, hash-deduped) and `activity` events as the activity ring appends. Every tab/view reflects new state within a beat without redundant polling. Polling continues at 8s as a safety net.

## Scope completed

* `HttpResponse` gains `streamMode` (bool) + `streamHandler` (callback). When `streamMode=true`, `SimpleHttpServer::handleClient` sends `Content-Type: text/event-stream` + `Cache-Control: no-cache` + `Connection: keep-alive` headers, then hands the SOCKET to the streamHandler on a detached worker thread so the accept loop never blocks on an SSE client.
* `handleClient` now closes the client socket explicitly on the non-streaming path too (pre-v0.9.71 sockets were left to OS recycling).
* `GET /api/events` streamHandler:
  * Initial `: mcos sse stream open\n\n` comment fires the EventSource `open` handler.
  * Every 1s tick: takes the current `DashboardSnapshot`, hash-dedupes against the previously-sent snapshot, and emits a `dashboard` event when the payload changed.
  * Same tick: reads `globalActivityRing().read(lastActivityId, 100)`; emits each new event as an `activity` event; advances the high-water mark.
  * `: ping\n\n` keepalive every 15 idle ticks so proxies / browsers don't time out the connection.
  * Honors both `serverRunning->load() == false` and `Impl::stopRequested_.load() == true` to exit cleanly on shutdown.
* `resources/web/app.js` `startRealtimeStream()` opens an EventSource on `/api/events`. Handlers:
  * `dashboard` → replace `state.dashboard`, call `renderCurrent()`.
  * `activity` → append to `state.activity.events` (capped at 500); re-render activity-bearing decks (overview/activity/telemetry).
  * `error` → the EventSource auto-reconnects.
* `#realtimeChip` in `resources/web/index.html` shows `LIVE` (SSE connected) / `POLLING` (fallback) / `CONNECTING` (initial state).
* Polling timer dropped from 2000ms to 8000ms (safety-net only; SSE handles real-time updates).

## Risks

* WinUI shell still polls `/api/dashboard` on its own 2s tick; it does not yet consume `/api/events`. A future iteration could share an EventSource via the shell's WebView control. Out of scope for this iteration; the real-time directive applies first to the browser dashboard which is the operator's primary surface.
* SSE `dashboard` events emit the full `DashboardSnapshot` per change. Hash-dedup avoids unchanged repeats but a busy snapshot still pushes ~30 KB per tick. A future iteration could push a delta instead.

## Deferred

WinUI shell minimize-to-tray + right-click stop/restart/close (v0.9.72), CLU model implementation against forsetti-agentic-edition governance (v0.9.73+).
