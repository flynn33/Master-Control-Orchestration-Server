# Release Completion Report — v0.9.89

Server-side kind filter on `/api/activity`.

## Scope completed

### `?kind=` query parameter

`/api/activity` previously took `?since=` + `?max=`. A consumer that wanted only a slice (audit panel showing supervisor events, watchdog dashboard showing pool failures, etc.) had to pull the full last-N events and filter client-side. On busy hosts this:

* Wastes bandwidth (the LAN client downloads the whole window every poll just to drop most of it).
* Loses earlier matches when N is small (last 50 events might be all `self_test`, and the supervisor event the operator wanted is just outside the window).

v0.9.89 adds a server-side filter:

* `?kind=supervisor_connect` → exact match.
* `?kind=supervisor_*` → prefix wildcard. Trailing `*` is the wildcard marker.
* When a filter is active, the ring read window expands to `min(kCapacity, maxCount * 4)` so a `?kind=supervisor_*&max=10` query doesn't return an empty array just because the last 10 events were all `self_test`.
* Response gains `kindFilter` echo field so consumers can confirm the server applied the filter.

Backward-compatible: queries without `?kind=` behave exactly as before (no filter, ring read window = maxCount).

### Live smoke

Against the v0.9.89 service immediately after deploy:

```
GET /api/activity?kind=supervisor_*&max=30
→ events:
   - supervisor_config_issue (200) → claude
   - supervisor_connect      (200) → claude
   - supervisor_disconnected (504) → claude    ← watchdog fired during the deploy gap!
→ kindFilter: "supervisor_*"
```

The third event (`supervisor_disconnected`, statusCode 504) is the supervisor watchdog firing in production: the v0.9.88 smoke test left a `connected` Claude supervisor, the v0.9.89 deploy stop/start gap exceeded the 120s heartbeat threshold, and `expireConnectionIfStale` flipped the state to `disconnected` once the new build started. The whole lifecycle is captured and now filterable.

## Public contracts changed

`/api/activity` response gains an optional `kindFilter` echo field (only present when the request supplied `?kind=`). The route remains backward-compatible: existing callers that don't supply `kind=` see no shape change.
