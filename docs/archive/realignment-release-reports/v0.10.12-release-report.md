# Release Completion Report — v0.10.12

Operator-directed eight-item punch list. The operator opened the session with a numbered list of bugs and feature gaps across the WinUI Shell and supervisor surface; v0.10.12 addresses every item in one release, rebuilds, deploys to the live spaces-path install, runs the validation chain, and smoke-tests the new endpoints end-to-end.

## 1. Action items (operator wording → resolution)

## 3. Public contracts changed

* **New routes:** `GET /api/chatgpt-plugin/status`, `POST /api/chatgpt-plugin/toggle`, `GET /api/grok-plugin/status`, `POST /api/grok-plugin/toggle`. Same JSON shape as the existing `/api/claude-plugin/status` + `/toggle` but the `registered` field reflects file presence at `<USERPROFILE>\Documents\MCOS\DirectAIControl\<provider>-mcos-control.json` instead of a directory junction. The toggle handler enforces mutual exclusion: turning ON any of the three slots forcibly revokes the other two (`removeClaudePluginJunction` + `setClaudeMcosPluginEnabled(false)` for the Claude slot, `removeDirectAIPluginConfig` for the json-drop slots).
* **New snapshot fields** on `/api/dashboard.mcpGatewayStatus`: no change. `mcpUrlRaw` was already exposed by v0.9.x. The Shell now consumes it.
* **New file artifact:** `<USERPROFILE>\Documents\MCOS\DirectAIControl\<provider>-mcos-control.json` carrying `mcos.direct-ai-plugin.v1` schema. Operator-facing artifact -- ChatGPT / Grok desktop don't natively read this path; the file is a copy-paste-ready snippet documenting the LAN-routable MCP gateway URL for manual import into the target client.

## 5. Risks and blockers

* **ChatGPT / Grok desktops don't natively read the connector config path.** The toggle drops `<USERPROFILE>\Documents\MCOS\DirectAIControl\<provider>-mcos-control.json` as an operator-facing artifact; neither ChatGPT desktop nor Grok desktop scans that directory automatically. The operator must copy `server.mcpEndpoint` into the target client's Custom Connector / Custom MCP UI manually. The instructions block in each connector file documents the path. A richer integration (publishing through ChatGPT Custom-GPT / Grok Connector APIs) is a v1.0.0+ candidate.
* **Cloud-hosted ChatGPT cannot reach LAN IPs.** The supervisor + direct-AI configs now carry LAN-routable URLs (v0.10.8 fix for supervisor; this release for direct-AI). A cloud runtime trying to reach `192.168.1.7` will still see `connection_refused`. The Supervisor card now has an explicit reachability note documenting the LAN-tunnel requirement (Tailscale / ngrok / Cloudflare Tunnel). Not a code fix; an operator-side network concern.
