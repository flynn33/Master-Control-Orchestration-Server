# Release Completion Report ‚Äî v0.9.68

External-dependency MCPs registered as default supervised pools. `docker-control` and `local-database` are now first-class catalog entries with a real in-tree worker ‚Äî the wire is live; the underlying capability (Docker Desktop / a configured DSN) is the operator concern, but the MCP route is reachable through the gateway and the tools surface configuration state honestly. ## Scope completed

* `Specialization::DockerControl` + `Specialization::LocalDatabase` + CLI parser branches.
* **docker-control**:
  * `docker.status` invokes `docker version --format json` via `runProcessCaptured`. `launchFailed` ‚Üí `dockerReachable=false` + actionable hint about installing Docker Desktop. Otherwise returns exitCode + raw daemon stdout/stderr.
  * `docker.list_containers` shells out to `docker ps --format {{json .}}` (or `--all` when `arguments.all=true`).
* **local-database**:
  * `db.status` reads `persistent-context['local-database.connection-string']`; reports `configured=true|false` honestly with hint text the operator can act on.
  * `db.set_connection_string` persists the connection string via `persistentCtxSet` so subsequent `db.*` tools (future iterations) can read it.
* `MasterControlRuntime` upserts 2 supervised pools.

## Risks

* `docker.list_containers` shells out via `docker.exe`. A future iteration could wire WinHttp directly to the Docker named pipe (`\\.\pipe\docker_engine`) for lower-overhead operation.
* `db.set_connection_string` persists the connection string to `%ProgramData%\MCOS\persistent-context.json` in plain JSON. Treat as machine-local secrets storage ‚Äî appropriate for the LAN-trust model but would need DPAPI wrapping for stronger posture.
