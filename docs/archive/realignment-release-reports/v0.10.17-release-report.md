# Release Completion Report — v0.10.17

POST-body unknown-key visibility on `/api/clients`, `/api/pools`, and `/api/telemetry/heartbeat`. The third item from the v0.10.15 debugger sub-agent scan. Pre-v0.10.17 a POST whose JSON body carried a field the destination model did not recognise (operator typo such as `enabledFlag` for `enabled`) returned HTTP 200 with the default value applied and zero indication anything had been dropped. v0.10.17 surfaces the dropped keys in an additive `droppedKeys` response array so operators see the diagnostic instead of debugging "why didn't my client enable?" from a clean 200.

The fourth POST route from the debugger note (`/api/runtime/mcp-servers`) calls `adminApiService_->upsertMcpServerJson(rawBody)` — it passes the raw string through, so the route layer never holds a typed model. Adding `droppedKeys` would require a route-layer parse before the service call; deferred to keep v0.10.17 scope tight. Captured in `knownIssuesDeferredToNextRelease`. Same call for `/api/clients/{clientId}/privileges`, `/api/runtime/subagents`, `/api/runtime/subagent-groups`.

## 5. Risks and blockers

- **Round-trip cost on every accepted POST.** Each refactored route now serialises the typed model back through `nlohmann::json()` to enumerate known keys. The cost is bounded by the model size (small for `LanClient`, `ManagedEndpointPool`, `ClientHeartbeat`) and only occurs on the POST path. Acceptable for this iteration; if a profiling pass surfaces it as hot, the known-key set could be memoised per type at static initialisation.
- **Top-level scoping.** The detector only inspects top-level keys. A nested object with an unknown field (e.g. `{privileges: {canFooBar: true}}`) is reported as the privileges key being known with no further detail. Nested traversal is a future iteration; the operator-facing typo cases are dominated by the top level today.
- **Partial coverage.** Three of seven similar POST routes were refactored. Parity for the remaining four (mcp-servers, privileges, subagents, subagent-groups) is captured in `knownIssuesDeferredToNextRelease`.
