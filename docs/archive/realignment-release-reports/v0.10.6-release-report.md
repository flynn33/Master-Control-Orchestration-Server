# Release Completion Report — v0.10.6

Iteration follow-up addressing the operator's explicit visual directive on the Telemetry deck of the WinUI Shell:

1. **The MCP Servers and Sub-Agents on the telemetry page MUST have smaller cards.**
2. **The MCP Server Cards MUST be close in size to the current Sub-Agent Cards from the overview page.**

The Overview deck's `SUB-AGENTS` card (`OverviewSectionControl.xaml.cpp::ApplySubAgentsCard`) is a single `ShellCardStyle` Border with thin text rows (`+ name` / `- name`) listed inside a `TextBlock`. There is no per-row Border, no per-row stroke, no per-row corner radius. Each row is just text on the parent panel's background. The Telemetry deck's compact cards (v0.9.99) still wrapped every row in its own TRON-red bordered Border with 8x4 padding and 8px corner radius, totaling ~28-32 vertical pixels per row. With ~28 MCP servers + 7 sub-agents on this host, the Telemetry stack occupied ~1100 vertical pixels and still drew visually heavier than the Overview SUB-AGENTS card.

## Scope completed

### Fix 1 — compact card density

`renderEndpointStatCardGrid()` in `EndpointStatCardGrid.h` now distinguishes its compact branch from the wide branch at the per-row container level:

* When `compact == true`, **the per-row Border is not created**. The row's `Grid` is appended directly to the parent StackPanel. The outer `ShellCardStyle` Border on the XAML side is the only frame; rows render flat inside it, matching the Overview SUB-AGENTS layout exactly.
* When `compact == true`, **the parent StackPanel's Spacing is forced to 2.0** from C++. Without this the XAML-declared `Spacing="10"` re-introduces the visual gap the operator flagged.
* Compact row padding shrinks `cardPadH 8.0 -> 6.0` and `cardPadV 4.0 -> 1.0`. With the Border gone, this padding is now a `Margin` on the row Grid so the reachability dots still line up with the parent panel's left edge.
* Compact name font shrinks from 12 to 11, util label font from 12 to 11, util ProgressBar height from 6 to 4.
* Wide (non-compact) rendering used by `RuntimeSectionControl` + `OverviewSectionControl` is unchanged.

Visible outcome on the Telemetry deck: 35 endpoints occupy ~700 vertical pixels instead of ~1100; the strip reads as a flat text list inside one bordered panel, mirroring the Overview SUB-AGENTS card.

### Fix 2 — SUB-AGENT GRID cross-tab footer hidden on Telemetry

Live screenshot of the Telemetry deck (post Fix 1 deploy) exposed the second half of the operator's regression report against v0.10.4: the v0.7.8 cross-tab `SUB-AGENT GRID` footer (`MainWindow::ApplySubAgentFooter` → `SubAgentFooterHeadline` + `SubAgentFooterGrid` in `MainWindow.xaml`) was still rendering as a separate row of bordered sub-agent tiles directly below the new Telemetry Sub-Agents thin-row card. That is why the operator reported "you have sub-agents in the telemetry view twice."

`MainWindow::ApplySubAgentFooter` now checks `currentDestination_ == kTelemetryDestination` first. On match it sets both `SubAgentFooterHeadline.Visibility` and `SubAgentFooterGrid.Visibility` to `Collapsed` and returns early without rebuilding any badges. Every other destination (Overview, Runtime, Imports, Exports, Security, Settings, CLU) continues to render the footer because those decks have no inline per-endpoint sub-agent panel.

## Risks and blockers

* The compact rendering still expects the operator to consult the Runtime tab or `/api/dashboard` to see per-card `endpointHostPort` / status word / active-client IP list. That information is intentionally hidden in compact mode to hit the density target.
* If the operator finds the row strip too dense (no visible separators), the next iteration can add a 1px hairline `Border.BorderBrush` along the row's bottom edge without re-introducing the v0.9.99 per-card stroke + corner radius.

## Deferred work

* Phase 2 of the CLU rebuild and Phase 3 (enforceAction wiring) still ship in v1.0.0+.
* Compact single-line Telemetry cards still hide per-card endpoint host:port + active-client list. v1.0.0+ candidate.
* Per-pass self-test rows in persistent Diagnostics log behind opt-in flag.
* PHASE-14 DiagnosticsSectionControl with FileSavePicker export, pending operator approval gate.
