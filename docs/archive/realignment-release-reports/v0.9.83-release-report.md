# Release Completion Report — v0.9.83

Operator-facing improvement on the generated supervisor config's `instructions` block.

## Scope completed

### Provider-specific instructions

The Supervisor Agent Assignment Wizard shipped in v0.9.76 produced a generated config with a generic 4-step instructions block:

```json
"instructions": {
    "summary": "Use this file on the LAN client that will act as the {provider} supervisor for MCOS.",
    "steps": [
        "Move this file to the selected supervisor client machine.",
        "Import or provide this file to the selected AI model/client.",
        "Connect to MCOS using the included endpoint and auth reference.",
        "Confirm supervisor connection when prompted."
    ]
}
```

That text was actionable for none of the three providers (Claude, ChatGPT, Grok) — the receiver-side import mechanics are very different per provider. v0.9.83 replaces the generic block with provider-specific guidance.

**Claude**: names the `~/.claude.json` / `.mcp.json` mcpServers field, names the Claude Desktop *Settings → Developer → Edit Config* path with `streamable-http` transport, optionally stores `auth.tokenRef` as `MCOS_SUPERVISOR_TOKEN_REF`. Each step inlines the live `server.mcpEndpoint` URL so the operator gets a copy-paste-ready value.

**ChatGPT**: names the Custom Connector / Custom MCP App registration path (ChatGPT macOS v1.2024.x+), names the `X-MCOS-Supervisor-ConfigId` header pattern, and surfaces the OAuth-edge fallback when the ChatGPT client doesn't yet support direct LAN MCP connections.

**Grok**: names the Grok MCP bridge / Grok-API-compatible supervisor client path. Honestly notes that a first-party Grok MCP client is not yet announced as of 2026-05; the config remains Grok-agnostic at the wire level (streamable-http MCP) so any future Grok bridge can use the file unchanged.

All three paths name the heartbeat watchdog cadence (~30s POST to `/api/supervisor/heartbeat`, 120s threshold) so the receiving client knows how to keep the assignment from flipping to `disconnected`.

### Test

`testSupervisorInstructionsAreProviderSpecific` runs three sub-cases (one per provider) and asserts each carries a marker phrase unique to its path: `"Claude Code"`, `"Custom Connector"`, `"Grok MCP bridge"`. Pre-v0.9.83 every provider's instructions were byte-identical, so the new test serves as a regression guard against a future generic-instructions slip.
