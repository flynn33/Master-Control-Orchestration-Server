## Scope completed

### `GET /api/governance/decisions` is self-describing

v0.9.19 wired the POST handler but a first-time client posting to `/api/governance/decisions` had to read the source (or stumble through `Unknown enum string` errors) to discover that `action` is `snake_case` (`remote_install`, `client_register`, ...) and not `PascalCase`. v0.9.20 turns the GET stub into a complete contract:

```json
{
  "method": "POST",
  "expects": "GovernanceEnforcementRequest",
  "returns": "GovernanceEnforcementDecision",
  "note": "POST /api/governance/decisions evaluates a request through CLU's enforceAction.",
  "validActionStrings": [
    "client_register", "client_privilege_change",
    "client_autonomous_mode_change", "client_revoke",
    "mcp_server_create", "mcp_server_modify", "mcp_server_remove",
    "sub_agent_create", "sub_agent_modify", "sub_agent_remove",
    "module_enable", "module_disable",
    "governance_policy_change", "remote_install"
  ],
  "possibleOutcomeStrings": ["allow", "block", "requires_operator_approval"],
  "requestShapeExample": {
    "action": "remote_install",
    "actor": "<client-id>",
    "subjectId": "<package-source-or-resource-id>",
    "allowUntrustedExecution": false
  },
  "responseShapeExample": {
    "action": "remote_install",
    "allowed": true,
    "outcome": "allow",
    "blockingFindings": [],
    "deferredActionId": "",
    "message": "",
    "posture": "<governance-posture>",
    "ruleId": ""
  }
}
```

The enum lists are built from the existing `to_string()` functions, so they stay in sync if the enum grows (currently 14 action values + 3 outcomes; the previous expansion from 2 to 15 happened in PHASE-07).
